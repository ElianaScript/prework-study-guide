HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Html attributes are always defined in the beginning element, never the closing element.
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage to users.
  • The title element defines the title for a webpage.
  • Search engine optimization is the process of improving the ranking of a website in search engines such as Google or Bing. Better SEO results in your website more frequently showing up in searches.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a website.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element enbeds client-side scripts in the HTML document.
  • The http:equiv sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.

Git

  • git status: checks what branch we are currently on.
  • git checkout: -b branch-name:creates a new branch and switches to it.
  • The git command must precede every git statement.
  • The add command adds modifications in the current working branch to the staging area.
  • The -A flag indicates that we want to add all changes.
  • The staging area informs Git that the changes will be added in the next commit.
